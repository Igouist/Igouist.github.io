Visual studio 環境筆記

2020-3-28,
IDE w3HexSchool
( ?  read )

s > 大綱：整理一下自己用的字型、插件和快捷鍵

今天在工作時寫到一半突然藍屏，重開機之後俺的地表最強編譯器 Visual Studio 大大整個就開始鬧脾氣，打開專案整排都是 Error，連 System.Object 都找不到，嚇到差點往生。還好用了修復功能之後一切恢復正常，但是載入的插件和一些個人設定就這樣重置了……

為了之後可能還會遇到相同的事情，這邊就先將平常的 Visual Studio 環境用到的設定做一份紀錄，之後遇到新插件或是什麼功能也可以回來更新這篇文，如此一來下次又被洗白的時候就可以回來參考了。

這邊主要會記錄三個部分，並隨時可能更新。主要是字型、使用的插件以及常用的快捷鍵。

不過我的 Visual Studio 也被洗白了所以這篇會跟著找回失落插件的歷程慢慢補上QQ

字型

字型部分必定、絕對要使用等寬字型！基本上來說，使用內建的 Consolas 就可以解決大半需求了。不過我這邊還要推薦兩個我常用的字型：

第一個是 FiraCode，整體看起來就是乾淨優雅，尤其是連字能使得整個 Code 更有滑順的感覺，十分推薦。

↓ FiraCode 的連字示意圖，取自 FiraCode 的 Github 頁面

然而，FiraCode 雖然陪伴我一段不短的時間，但還是有一個致命的缺陷：不支援中文！於是後來我基本上都使用推薦的第二款：更紗黑體

除了同樣支援連字以外，更支援多國語系。例如繁中就是有標註 TC 的字形，看見中英文都套用上去就是一陣舒服。

↓ 更紗黑體，圖片取自微軟 Store

插件

CodeMaid

極實用，光是可以設定儲存時自動排版就拯救了我這個懶人。至於方法的複雜度部分我都是參考而已。

Editor Guidelines

可以在指定字數劃出直的輔助線，我都用來標示出 100 字元 和 120 字元的位置，用來提醒自己換行。

快捷鍵

這邊大概整理一下平常會按到的快捷鍵，方便上班在外可以直接回來開起來看。希望有朝一日能夠寫起來時雙手不離鍵盤 XD

偵錯

  • 執行(debug)：F5
  • 執行(non debug)：Ctrl + F5
  • 全部儲存：Ctrl + Shift + S
  • 單步執行：F11 (F10) :star:
  • 切換斷點：F9 :star:
  • 啟用/停用斷點：Ctrl + F9
  • 刪除所有斷點：Ctrl + Shift + F9
  • （單元測試）對全部測試：Ctrl + R, A
  • （單元測試）對全部偵錯：Ctrl + R, Ctrl + A

檢視

  • 程式碼編輯器分頁切換 ：Ctrl+TAB :star:
  • 顯示屬性窗口：F4
  • 關閉目前視窗：Ctrl + F4
  • 移至定義：F12 :star:
  • 列出參考：Shift + F12 :star:
  • 移至目標：Ctrl + T => 輸入目標 :star::star::star:
  • 巡覽列：Ctrl + F2 => Tab
  • 工具列：Alt + (對應鍵)
  • 回到編輯器區塊：F7

編輯

  • 選取目前文字：Ctrl + W
  • 選取同個變數：Shift + Ctrl + ↑/↓
  • 選取目標區塊：Shift + Ctrl + }
  • 註解選取範圍：Ctrl + K,C
  • 取消註解選取範圍：Ctrl + K,U
  • 選取文字改小寫：Ctrl + U
  • 選取文字改大寫：Ctrl + Shift + U
  • 呼叫出類別成員：Ctrl + J (編到一半時 tips 突然不見很好用)
  • 收攏原始碼：Ctrl + M, O
  • 展開原始碼：Ctrl + M, L
  • 收攏／展開 Region：Ctrl + M, Ctrl + M
  • 刪除目前這行：Ctrl + Shift + L
  • 刪除目前往後：Ctrl + Delete
  • 檔案最前面：Shift + Home
  • 檔案最後面：Shift + End
  • 選到最前面：Ctrl + Shift + Home
  • 選到最後面：Ctrl + Shift + End
  • 跳至行號：Ctrl + G :star:
  • 在上面插入一行： Ctrl + Enter
  • 多行選取：Shift + Alt + ↑ or ↓ :star:
